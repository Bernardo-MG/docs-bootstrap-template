Bootstrap Template and Demo for a Project's Documentation

Taking modern features of Bootstrap and HTML5, this template aims to give a modern and responsive web design to documentation sites, so they can be read easily from any kind of device.

It's meant to be used for any kind of project, and so it is generic and reusable, easy to adapt for any concrete needs, and to any template engine or documentation framework, such as Jinja, Maven Site or Sphinx.

Usage

Just grab the template from GitHub and adapt the html files with your preferred templating engine.

All the static files, which are required for the template, are stored in the _static folder.

Features

  • Simple style, avoiding unneeded complexities.
  • HTML5 semantic markup.
  • Semantic tags for Facebook OpenGraph and Twitter Cards.
  • Bootstrap used for a responsive design which adapts to the device being used.
  • Font Awesome for the icons.
  • highlight.js for code highlighting.
  • Validated for HTML 5, following the W3C guidelines.
  • Validated for accessibility, following the WCAG 2.0 (Level AA) guidelines, and also helped by Wave.
  • Validated CSS, following the W3C and Twitter recess guidelines.
  • Validated JS.
  • Additionally it is being continuously validated with other tools through Travis CI.

Examples

To illustrate the template several pages have been created. These are divided into two groups, first there are some pages showcasing different uses of the template, which serve also smaller subtemplates, and there are also some test pages, for checking that the template works as expected.

General use cases:

  • Tests report
  • Changes log
  • Code documentation

Tests to check the UI:

  • Components
  • A page with minimal content

In all of these pages you can always go back to the index by clicking the project's name on the navigation bar.
